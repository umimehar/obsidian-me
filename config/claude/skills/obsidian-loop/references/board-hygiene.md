# Board hygiene selectors â€” what each mode flags

Detail for the six read-only audit selectors. The SKILL.md hot path keeps the runnable command block; these per-flag explanations load on demand. Each mode prints a JSON array that is empty when the board is clean.

- `--lint` flags tickets pinned to a registered device whose `repos` do not cover the ticket's project (a pin nobody can ever claim).
- `--schema` flags tickets missing required frontmatter fields and invalid `orchestrator/control.md` values (a bad `loop`, `max_effort`, `allowed_types` entry, or `max_tickets_per_day`).
- `--coverage` flags `ready` non-`system` tickets whose `project` appears in NO device's `repos` â€” work no device in the fleet can ever claim until a device adds the repo.
- `--card-presence` flags ticket notes with no card on their board (`missing_card`) and board cards with no backing ticket note (`orphan_card`). **Board scope:** `system` tickets belong on `orchestrator/board.md`; every other ticket belongs on `projects/<project>/board.md` per its frontmatter `project` (a missing project board counts as a missing card). Every status maps to a column, so every well-formed note must have exactly one card.
- `--status-drift` flags a ticket whose card sits in a board column that disagrees with its frontmatter `status`, using the Columnâ†”status map (`backlogâ†’Backlog`, `readyâ†’Ready`, `claimedâ†’Claimed`, `in-progressâ†’In Progress`, `reviewâ†’Review`, `blockedâ†’Blocked`, `doneâ†’Done`, and `failedâ†’Blocked` â€” a failed ticket keeps `status: failed` but parks in the Blocked column). This is the complement to `--card-presence`: card-presence checks a card *exists*, status-drift checks it is in the *right* column. Frontmatter is the source of truth, so reconcile the card to the status â€” unless a newer deliberate human drag is the real signal, in which case reconcile the status to the moved card (use `git log`/timestamps to see which side changed last; a `vault backup:` commit moving a card is the Obsidian Git plugin capturing a human drag). A missing card is left to `--card-presence`; an unknown status is left to `--schema`.
- `--stale` flags `claimed`/`in-progress`/`review` tickets whose owning device (`claimed_by`) has a `last_heartbeat` older than the threshold (default 24 h, override with `--hours N`). A `claimed_by` of `null`/`any` or an unregistered slug cannot be resolved to a device note; those tickets are reported with a reason (never skipped silently, never a crash).
