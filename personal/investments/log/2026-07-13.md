# 2026-07-13

Scaffolded the investments endeavor and the CSV to JSON to HTML pipeline.

Built the CLI orchestrator (`scripts/build.py`) that wires together parse, classify, mask, datastore, analytics, and render into one run, and added an end to end write test to `tests/test_datastore.py`.

Ran the pipeline against the real Wealthsimple statements in `~/Downloads/monthly-statements-2022-01-to-2026-07` using a local gitignored `redactions.json` (not committed) built from `redactions.example.json`. The build produced 8050 transactions across 18 accounts from 183 source files, with the row count reconciliation passing (parsed count matched stored count). All six HTML pages were written to `notes/` (index, cash-flow, contributions, growth, holdings, income) alongside `data/datastore.json` and `data/analytics.json`.

Verified privacy after the real build. A grep for the real account code pattern (`WK`/`HQ`/`WZ` followed by nine alphanumerics) across `personal/investments/` returned zero matches. Each real name in the local redaction list was grepped against the built data and HTML pages with zero matches. E-transfer style description fields were spot checked and read `[redacted]` rather than a name.

Full test suite (43 tests), ruff check, ruff format check, and ty check all pass clean. No unmapped transaction codes were reported by the real build.
